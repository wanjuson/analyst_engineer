CH.2 데이터 분석 기획
2절. 분석방법론
 1.분석 방법론 개요
 가.개요
  - 체계화한 절차와 방법이 정리된 데이터 분석 방법론 수립이 필수
   . 방법론은 상세한 절차, 방법, 도구와 기법, 플릿과 산출물 필요
 나.데이터 기반 의사결정의 필요성
   1)경험과 감에 따른 의사결정 -> 데이터 기반의 의사결정
   2)기업의 합리적 의사결정을 가로막는 장애요소 : 고정관념, 편향된 생각, 프레이밍 효과
 다.방법론의 생성과정
  - 암묵지 : 학습과 경험을 통해 개인에게 체화되어 있지만 겉으로 드러나지 않은 지식 / 공통화, 내면화
  - 형식지 : 문서나 매뉴얼처럼 형상화된 지식 / 표출화, 연결화
 라.방법론의 적용 업무의 특성에 따른 모델
  1)폭포수 모델 : 단계를 수차적으로 진행하는 방법론
  2)프로토타입 모델 : 일부분을 우선 개발하고 그 결과를 통한 개선 작업 시행
  3)나선형 모델
 마.방법론의 구성 : 단계 > 태스크 > 스탭
 
 2.KDD 분석 방법론
 나.KDD 분석 절차
  1)데이터셋 선택 : 비즈니스 도메인에 대한 이해, 프로젝트 목표 설정, 목표데이터 설정
  2)데이터 전처리 : 잡음, 이상치, 결측치를 식별하고 필요시 제거
  3)데이터 변환 : 데이터의 차원을 축소, 학습용 데이터와 검증용데이터로 데이터를 분리
  4)데이터 마이닝 : 데이터마이닝 기법을 선택, 데이터 전처리와 데이터 변환 프로세스를 추가로 실행하여 최적의 결과를 산출
  5)데이터 마이닝 결과 평가 : 데이터마이닝 결과에 대한 해설과 평가, 분석 목적과의 일치성 확인.
    분석결과를 업무에 활용
 
 3.CRISP-DM
 가.개요 : 계층적 프로세스 모델
 다.프로세스
 1)업무이해 : 프로젝트의 목적과 요구사항 이해, 데이터 분석을 위한 문제정의
 2)데이터이해 : 데이터를 수집하고 데이터 속성을 이해, 인사이트를 발견
 3)데이터준비 : 분석을 위하여 수집된 데이터에서 분석기법에 적합한 데이터를 편성하는 단계
 4)모델링 : 다양한 모델링 기법과 알고리즘을 선택, 최적화
    모델평가를 통해서 모델의 과적합 문제를 확인
 5)평가 : 모델링 결과가 프로젝트 목적에 부합하는지 평가

ㅁ 맵리듀스
 - 맵이듀스에서 블록 크기의 기본값은 64MB이며 맵 태스크가 1대의 블록을 대상으로 연산을 수행한다.
 
 ㅁ ODS
  - 데이터가 스테이징 단계에서 타임스탬프, 데이터 값에 대한 체크 섬 등의 통제 정보들이 추가될 수 있다.
  - 데이터 스테이징 단계에서는 정기적인 ETL과 실시간 ETL을 혼용할 수 있고, 저장되는 테이블의 스키마는 데이터 원천의 구조에 의존한다.
  - 데이터 원천과 스에티징 테이블과의 매핑은 일대일 혹은 일대다로 구성될 수 있다.
   
 ㅁ HDFS
  - 데이터 랜덤 접근 방식을 지원하지 않으며 순차 접근 방식을 지원하다.
  - 구글 파일 시스템의 아키텍처와 사상을 그대로 구현한 클로닝 프로젝트이다.
  - 데이터를 블록이나 청크 단위로 저장한다.
  - 기본적으로 파일은 한번 쓰이면 변경되지 않는다고 가정한다.
 
 ㅁ 하둡에코시스템에서 사용되는 기술
  - 데이터 수집 기술 : Flume, kafka, sqoop
  - 대용량 SQL 질의 기술 : Hive, Pig, Sqoop
  - 워크플로 관리 : Oozie, Azkaban
  - 실시간 SQL 질의 기술 : Impala, Tajo
