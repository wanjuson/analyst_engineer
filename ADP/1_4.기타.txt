CH.1 데이터 이해
기타. 최신 빅데이터 상식
 1.DBMS와 SQL
  - DBMS : 데이터베이스를 공유하며 사용할 수 있는 환경을 제공하는 소프트웨어
  - 관계형 DBMS : 각각의 항목과 그 속성이 다른 모든 항목 및 그의 속성과 연결될 수 있도록 구성된 자료 구조
  - 객체형 DBMS : DATA를 '객체' 형태로 표현하는 데이터베이스 모델
  - 네트워크 DBMS : 레코드들이 노드로, 레코드들 사이의 관계가 간선으로 표현되는 그래프를 기반으로 하는 데이터베이스 모델
  - 계층형 DBMS : 트리 구조를 기반으로 하는 계층 데이터베이스 모델
  - Structured Query Language: SQL
    데이터베이스 시스템에서 자료를 처리하는 용도로 사용되는 구조적 데이터 질의 언어
	
 2.Data에 관련한 기술
 가.개인정보 비식별 기술
  - 데이터 마스킹 : 데이터의 길이, 유형, 형식과 같은 속성을 유지한채, 새롭고 읽기 쉬운 데이터를 익명으로 생성하는 기술.
  - 가명처리 : 개인정보 주체의 이름은 다른 이름으로 변경하는 기술, 다른 값으로 대체할 시 일정한 규칙이 노출되지 않도록 주의해야함.
  - 총계처리 : 데이터의 총합값을 보여줌으로써 개별 데이터의 값을 보이지 않도록 함
  - 데이터값 삭제 : 데이터 공유, 개발 목적에 따라 데이터 셋에 구성된 값 중에 필요 없는 값 또는 개인식별에 중요한 값을 삭제.
  - 데이터 범주화 : 데이터의 값을 범주의 값으로 변환하여 값을 숨김.
 
 나.무결성과 레이크
  1) 데이터 무결성 : 데이터에 대한 일관성, 유효성, 신뢰성을 보장하기 위해 데이터 변경 수정시 제한을 두어 데이터의 정확성을 보증함
   A.개체 무결성 :  기본키를 구성하는 모든 속성은 널 값을 가지면 안 된다는 규칙
   B.참조 무결성 : 외래키 값은 피참조 릴레이션의 기본키 값이거나 NULL 값임
   C.범위 무결성(Domain integrity) : 범위 무결성은 정의된 범위에서 관계형 데이터베이스의 모든 열이 선언되도록 규정
   
 * 데이터의 유형
  - 정형데이터 : 형태가 있으며 연산이 가능함.
  - 반정형데이터 : 형태가 있으며 연산이 불가능. 주로 파일로 저장됨
  - 비정형데이터 : 형태가 없으며 연산이 불가능. 주로 NoSQL에 저장됨
